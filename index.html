Zmienne

Zmienne w PHP służą do przechowywania danych. Nazwa zmiennej musi rozpoczynać się znakiem dolara $ i może zawierać litery, cyfry oraz znaki podkreślenia. 

$zmienna = "Wartość";
    $liczba = 42;
    $tablica = array("PHP", "JavaScript", "Python");

Zmienne w PHP nie wymagają deklaracji typu danych, ponieważ PHP jest językiem słabo typowanym.

Funkcje

Funkcje to bloki kodu, które wykonują określone zadania. Można je definiować i wywoływać w dowolnym miejscu w kodzie.

Definiowanie funkcji

function nazwaFunkcji($parametr1, $parametr2) {
        // kod funkcji
        return $wynik;
    }

Funkcja może przyjmować dowolną liczbę parametrów i zwracać wartość za pomocą instrukcji return.

Wywoływanie funkcji

$wynik = nazwaFunkcji($argument1, $argument2);

Funkcja jest wywoływana przez podanie jej nazwy i przekazanie odpowiednich argumentów.

Funkcje anonimowe

$funkcjaAnonimowa = function($x, $y) {
        return $x * $y;
    };
    echo $funkcjaAnonimowa(3, 4); // 12

Funkcje anonimowe (tzw. closure) nie mają nazwy i mogą być przypisywane do zmiennych.

Pętle

Pętle pozwalają na wielokrotne wykonywanie tego samego bloku kodu. Najczęściej używane pętle w PHP to for, while, do-while oraz foreach.

Pętla for

for ($i = 0; $i < 5; $i++) {
    echo $i . "<br>";
}

Pętla for składa się z trzech części: inicjalizacji, warunku i iteracji. Wykonuje się dopóki warunek jest prawdziwy.

Pętla while

$i = 0;
while ($i < 5) {
    echo $i . "<br>";
    $i++;
}

Pętla while wykonuje się dopóki warunek jest prawdziwy.

Pętla do-while

$i = 0;
do {
    echo $i . "<br>";
    $i++;
} while ($i < 5);

Pętla do-while wykonuje się co najmniej raz, a następnie kontynuuje dopóki warunek jest prawdziwy.

Pętla foreach

$colors = array("red", "green", "blue");
foreach ($colors as $color) {
    echo $color . "<br>";
}

Pętla foreach służy do iterowania po tablicach lub obiektach.

Instrukcja if

Instrukcja if pozwala na wykonywanie kodu tylko wtedy, gdy spełniony jest określony warunek. 

$x = 10;
if ($x > 5) {
    echo "Większe niż 5";
} else {
    echo "Mniejsze lub równe 5";
}

Można również użyć instrukcji elseif do sprawdzenia wielu warunków.

Instrukcja switch

Instrukcja switch pozwala na wybór jednego z wielu bloków kodu do wykonania, w zależności od wartości wyrażenia. 

$dzien = 2;
switch ($dzien) {
    case 1:
        echo "Poniedziałek";
        break;
    case 2:
        echo "Wtorek";
        break;
    default:
        echo "Inny dzień";
}

Instrukcja break powoduje wyjście z instrukcji switch po wykonaniu odpowiedniego bloku kodu.

Łączenie się z bazą danych

W PHP, aby połączyć się z bazą danych, należy użyć odpowiednich funkcji, takich jak mysqli_connect() lub PDO. 

$servername = "localhost";
$username = "username";
$password = "password";
$dbname = "database_name";

// Tworzenie połączenia
$conn = new mysqli($servername, $username, $password, $dbname);

// Sprawdzenie połączenia
if ($conn->connect_error) {
    die("Połączenie nie powiodło się: " . $conn->connect_error);
}
echo "Połączono z bazą danych!";

Pobieranie danych z formularzy

Aby pobrać dane z formularza HTML, można użyć superglobalnych zmiennych $_GET lub $_POST, w zależności od użytej metody. 

<form method="post" action="<?php echo $_SERVER['PHP_SELF'];?>">
  Imię: <input type="text" name="imie"><br>
  Nazwisko: <input type="text" name="nazwisko"><br>
  <input type="submit" name="submit" value="Wyślij">
</form>

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $imie = $_POST['imie'];
    $nazwisko = $_POST['nazwisko'];
    echo "Witaj, " . $imie . " " . $nazwisko . "!";
}
?>

Funkcja isset()

Funkcja isset() służy do sprawdzania, czy zmienna została zainicjowana i nie jest null. 

<?php
$zmienna = "Wartość";
if (isset($zmienna)) {
    echo "Zmienna jest zainicjowana";
} else {
    echo "Zmienna nie jest zainicjowana";
}
?>

Metody GET i POST

GET i POST to dwie główne metody przesyłania danych w formularzach HTML.

  • GET: Dane są dołączane do adresu URL i widoczne w pasku przeglądarki. Zalecane dla mniejszych ilości danych.
  • POST: Dane są przesyłane w treści żądania HTTP i nie są widoczne w adresie URL. Zalecane dla większych ilości danych lub poufnych informacji.
